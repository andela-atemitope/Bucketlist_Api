    Introduction

    Bucketlist_API is a Restful helpful APi that helps create a bucketlist which supports the addition of multiple items to the bucketlist and also the ability to tick an item as done(completed or incompleted). This is a detailed documentation on how to consume this API and better interact with its end points

    Creating an Account

    Creating an account requires that you make a post request to the user/new endpoint and passing following parameters as part of the

    To create an account send a POST request containing the necessary params to

    # With shell, you can just pass the correct header with each request
localhost:3000/api/v1/users?username="youruserame"&email=janedoe@email.com&password=password

    The above command returns JSON structured like this:

This token is used to authenticate your requests and it has to be passed in the headers along with every request when making other operations with this api. By default, each token has 24 hours of validity. If your token expires you should generate a new one by following the above steps

    ERRORS If you try using your token after 24 hours, you would get a JSON response saying “error: your token has expired”. Once presented with this error, you should follow the above steps to generate another token

    Ensure you copy your token correctly as you will be presesnted with a JSON response saying ‘error: invalid token’. It is advised that you save your token correctly

    AUTHENTICATION

    The authentication method used by a bucketlist_api is via the use of tokens and its being used throughout the API’s operations

    # In the header of the requests, the token should be passed as shown below

Authorization: Token qij0j0909y89gibnkihojibojniubknjsdnsdoj

    After an account has been created and the user logs in, a unique token is generated and assigned to the account. This token is passed in the header when performing any of the operations that require authentication.This generated token is valid for 24 hours and expires afterwards. Upon expiry the user should login (See above) to generate a new token

    The token should be included in the header in the following format

    Authorization: Token qij0j0909y89gibnkihojibojniub

    Your token expires after 24 hours, hence you would be required to generate another

    THE BUCKETLIST

    Creating a Bucketlist

    curl -X POST -d "http://example.com/api//2"
  -H "Authorization: Token dfijdiojfdjofjdjodjfoqwnooniomn"

    The above command returns JSON structured like this:

    
{  "SUCCESS" : "Tutorial Bucketlist created!" }

    You can create a new bucketlist by making a post request to the api/v1/bucketlists endpoint and passing in the neccesary parameters in the request

    These parameters include the bucketlist name(the desired name for the bucketlist), passed in the request and the authentication token passed in the header

    HTTP Request

    POST http://bucketlist_api/api/v1/bucketlists

    Query Parameters

    Parameter Default Description
    name nil This is the desired name for the bucketlist
    token nil This is used to authenticate the user against
    If any of these parameters are not provided you will be presented with the appropriate error

    Adding Items to a Bucketlist

    You can add items to a bucketlist by making a post request to the /api/v1/bucketlist/<id>/items endpoint and passing in the id and the name in alphanumeric characters with the request and the authentication token in the header 

    curl -X POST -d  "name=Item1"
             -H  "Authorization: Token token=thisiswhereyouwillputyourtoken"

    The above command returns JSON structured like this:

    Editing Bucketlists

    Edit a Bucketlist

    curl-X PUT -d "name=newbucketlistname"
           -H "Authorization: Token token=entertokenhere"
           "https://bucketlist_api.herokuapp.com/api/v1/bucketlists/69"

    You can edit a bucketlist by making a put request to the /api/v1/bucketlists/ endpoint and passing in the neccesary parameters with the request

    Query Parameters

    Parameter Default Description
    name nil This is the new_name you wish to assign to the bucketlist
    id nil This is the id of the bucketlist you intend to edit

    Delete a Bucketlist

    curl-X DELETE -d "id=69"
              -H "Authorization: Token token=insertyourtokenhere"
              "https://bucketlist_api.herokuapp.com/api/v1/bucketlists/69"

    The above command return JSON structured like this: 

    {
  "message": "Bucket list and all its associated items deleted"
}

    Deleting a bucketlist can be done by making a DELETE request to the /api/v1/bucketlists/<id> endpoint and passing the neccesary parameters

    You will be required to pass in the authentication token in the header with this request

    HTTP

    DELETE https://bucketlist_api.herokuapp.com/api/v1/bucketlists/69

    Delete a Bucketlist Item

    curl-X DELETE -d "bucketlist_id=3 item_id=69"
              -H "Authorization: Token token=insertyourtokenhere"
              "https://bucketlist_api.herokuapp.com/api/v1/bucketlists/3/items/69"

    The above command return JSON structured like this: 

    {
  "message": "Item has been successfully deleted"
}

    Deleting a bucketlist can be done by making a DELETE request to the /api/v1/bucketlists/<bucketlist_id>/items/<item_id> endpoint and passing the neccesary parameters

    HTTP

    DELETE https://bucketlist_api.herokuapp.com/api/v1/bucketlists/3/items/69

    Pagination

    Fetching a paginated response is done by sending a GET request passing the limit and page parameters to the /api/v1/bucketlists endpoint. Not more than 100 records are displayed if the value of the limit is greater than 100 and not less than 20 records are displayed unless the amount of responses are less than 20.

    ENDPOINTS

    Below is the summary of the api’s endpoints, its functionality and its public access .

    TBucket API has the following Endpoints:

    EndPoint Public Access Functionality
    POST true /auth/login – Logs a user in
    GET false /auth/logout – Logs a user out
    POST false /bucketlists/ – Create a new bucket list
    GET false /bucketlists/ – List all the created bucket lists
    GET false /bucketlists/ – Get single bucket list
    PUT false /bucketlists/ – Update this bucket list
    DELETE false /bucketlists/ – Delete this single bucket list
    POST false /bucketlists//items/ – Create a new item in bucket list
    PUT false /bucketlists//items/ – Update a bucket list item
    DELETE false /bucketlists//items/ – Delete an item in a bucket list
